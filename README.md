## 🎯 О проекте

Данный проект представляет собой полностью автономную систему компьютерного зрения на базе микроконтроллера STM32H750 с использованием TensorFlow Lite Micro. Система выполняет инференс нейронной сети непосредственно на микроконтроллере без подключения к компьютеру или облачным сервисам. Видеопоток захватывается с камеры OV2640, обрабатывается и подается на вход нейросети, а результаты классификации отображаются на LCD дисплее ST7735 в реальном времени. Ключевая особенность проекта — полная автономность и работа с частотой 10 кадров в секунду.

---

## 🛠 Аппаратная платформа

**Микроконтроллер:** STM32H750VBT6 на ядре Cortex-M7 с тактовой частотой 400 МГц. Устройство имеет 128 КБ внутренней Flash и 1 МБ SRAM, однако для размещения кода приложения и модели используется внешняя память.

**Камера:** OV2640 с разрешением 2 мегапикселя, подключенная через интерфейс DCMI (Digital Camera Interface) для захвата видеопотока и I2C для конфигурации регистров. Тактовый сигнал XCLK 12 МГц подается с микроконтроллера.

**Дисплей:** ST7735 с диагональю 0.96 дюйма и разрешением 160×80 пикселей, подключенный по SPI4. Используется для отображения видеопотока и результатов классификации.

**Внешняя память:** W25Q64 объемом 8 МБ с интерфейсом QSPI, в которой размещается код приложения и данные модели.

**Отладка:** UART1 со скоростью 921600 бод для вывода отладочной информации и результатов инференса.

## 🧠 Архитектура программного обеспечения

Проект построен по модульному принципу с четким разделением на независимые компоненты:

**Модуль ProcessModel** отвечает за интеграцию TensorFlow Lite Micro. Он содержит всю логику работы с нейросетью: инициализацию интерпретатора, загрузку модели, выделение тензорной памяти, выполнение инференса и обработку результатов. Модель квантована в формат uint8 и занимает 14744 байта. Входной тензор имеет размер 32×32×3, выходной содержит вероятности для 4 классов.

**Модуль Camera** реализует драйвер камеры с автоматическим определением типа подключенного устройства. Поддерживаются камеры OV7670, OV2640, OV7725 и OV5640. Основная камера проекта — OV2640 с I2C адресом 0x60. Модуль обеспечивает конфигурацию разрешения, формата пикселей и генерацию тактового сигнала XCLK.

**Модуль LCD** предоставляет функции для работы с дисплеем ST7735 через SPI интерфейс. Реализован вывод текста с использованием встроенных шрифтов 12×6 и 16×8 пикселей, отрисовка геометрических примитивов, управление подсветкой через аппаратный или программный ШИМ, а также функции наложения текста на видеопоток с прозрачным или непрозрачным фоном.

**Модуль UART** обеспечивает отладку через последовательный порт. Реализован форматированный вывод с поддержкой всех спецификаторов printf, отправка строк и бинарных данных. Все функции работают в блокирующем режиме с таймаутом 1000 мс.

## 🔄 Конвейер обработки данных

Захват изображения с камеры выполняется через DCMI с использованием двойной буферизации (Ping-Pong режим). DMA автоматически переключает буферы при заполнении, обеспечивая непрерывный захват без потери кадров. При завершении захвата формируется прерывание, и флаг готовности кадра устанавливается для основного цикла.

Из готового буфера камера отдает изображение в формате RGB565 с разрешением 160×120 пикселей. Для подачи на вход нейросети требуется изображение 32×32×3 в формате RGB888. Конвертация выполняется по специально разработанному алгоритму с предварительно рассчитанными картами масштабирования. Карты рассчитываются один раз при первом кадре и содержат соответствие между координатами исходного и результирующего изображений. Это позволяет выполнять преобразование без вычислений с плавающей точкой на каждом пикселе, сокращая время конвертации до 0.8 миллисекунды.

После конвертации данные копируются во входной тензор интерпретатора TFLite Micro с учетом типа квантования. Для модели с uint8 используется прямое копирование, для int8 выполняется преобразование вычитанием 128. Запуск инференса осуществляется вызовом метода Invoke интерпретатора, после чего из выходного тензора извлекаются вероятности классов.

Обработка результатов включает поиск класса с максимальной уверенностью, сохранение всех вероятностей для статистики, обновление счетчиков и визуализацию. На LCD дисплее отображается цветная рамка, соответствующая распознанному классу: белая для пустого кадра, синяя для круга, красная для квадрата, зеленая для треугольника. Также выводятся название класса, уверенность в процентах и время выполнения инференса в миллисекундах.

## 📊 Модель компьютерного зрения

Используется квантованная модель TensorFlow Lite размером 14744 байта с входным тензором 32×32×3 и выходным тензором для 4 классов. Модель обучена распознавать следующие объекты: пустой фон, круг, квадрат и треугольник. Квантование в формат uint8 позволяет эффективно использовать ресурсы микроконтроллера и выполнять инференс за 80-120 миллисекунд.

**Параметры модели:**
- Размер: 14744 байта
- Версия схемы TFLite: 3
- Тип квантования: uint8
- Вход: 32×32×3 RGB
- Выход: 4 класса

**Классы:**
- ID 0: EMPTY (пусто/фон) — белая рамка
- ID 1: CIRCLE (круг) — синяя рамка
- ID 2: SQUARE (квадрат) — красная рамка
- ID 3: TRIANGLE (треугольник) — зеленая рамка

## ⚡ Производительность системы

**Временные характеристики:**
- Захват кадра через DMA: менее 1 миллисекунды
- Конвертация 160×120 → 32×32: 0.8 миллисекунды
- Инференс модели: 80-120 миллисекунд
- Вывод на LCD: 2-3 миллисекунды
- Полный цикл обработки: около 100 миллисекунд (10 кадров в секунду)

**Распределение времени выполнения:**
- Инференс нейросети: 95%
- Конвертация изображения: 1%
- Вывод на LCD: 3%
- Прочие операции: 1%

## 💾 Особенности работы с памятью

Проект использует внешнюю QSPI Flash W25Q64 объемом 8 МБ для хранения кода приложения и данных модели. Это необходимо, так как внутренней Flash микроконтроллера STM32H750 объемом 128 КБ недостаточно для размещения полного проекта с TensorFlow Lite Micro.

Программирование внешней Flash требует специального подхода, поскольку стандартный программатор ST-Link не может напрямую прошить W25Q64. Используется загрузчик, который прошивается во внутреннюю Flash через ST-Link, после чего основное приложение загружается через UART.

## 📟 Отладка и мониторинг

Для отладки используется UART1 со скоростью 921600 бод. В процессе работы выводятся следующие данные:

- Информация о инициализации системы и обнаруженной камере
- Параметры модели и размер тензорной арены
- Время выполнения каждого инференса
- Распознанный класс и уверенность в процентах
- Статистика производительности

Пример вывода:

SYSTEM STARTING...
Camera found: OV2640 (ID 0x2642)
STARTING INFERENCE #1 - completed in 95 ms
RESULT: CIRCLE (1), Conf: 210/255 (82%), Time: 95 ms
PERFORMANCE STATS: Current:95 Avg:95 Min:95 Max:95

## 📁 Структура проекта

Проект организован следующим образом:

В папке Core размещаются основные файлы приложения: заголовочные файлы main.h, use_main.h, camera.h, lcd.h, uart_use.h и исходные файлы main.c, use_main.c, common_use.c, camera.c, lcd.c, uart_use.c.

В папке ProcessModel находится модуль TensorFlow Lite Micro с файлами ProcessModel.h, ProcessModel.cpp и model_uint8.h, содержащим квантованную модель.

Папка Drivers содержит драйверы периферии, Middlewares — библиотеку TensorFlow Lite Micro, docs/images — изображения для документации.

## 🚀 Применение

Проект может служить основой для создания встраиваемых систем компьютерного зрения: умных камер видеонаблюдения с детекцией объектов, систем сортировки на конвейере, роботов с техническим зрением, устройств распознавания жестов и интерфейсов человеко-машинного взаимодействия.

Благодаря использованию TensorFlow Lite Micro система легко адаптируется под другие задачи путем замены модели и минимальных изменений в коде обработки результатов.
